## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱명 | 사주광 (四柱狂) |
| 플랫폼 | 토스 미니앱 (웹뷰 기반) |
| 수익 모델 | 기본 무료 + 토스 내 광고 |
| 기술 스택 | React + TypeScript (프론트) / Node.js + TypeScript (백엔드) |
| DB | Supabase (PostgreSQL) |
| AI | Google Gemini API (2.5 Flash) |

### 핵심 차별점
- **태양시(지역시) 보정 적용** — 국내 대부분의 사주 앱과 달리 실제 태양 위치 기준으로 계산 (서울 기준 표준시 -32분 보정)
- **만세력 엔진** — KASI(한국천문연구원) 데이터 기반 `@fullstackfamily/manseryeok` 라이브러리 사용
- **AI 개인화 해석** — DB 사주 계산값을 Gemini AI에 전달해 개인 맞춤 해석 생성

## 2. 앱 전체 구조

### 2-1. 화면 플로우
```
온보딩 (최초 1회)
└── 생년월일시 + 성별 입력 → 사주 계산 API 호출

내 사주 홈 (메인)
├── 나의 기운 (메인 오행 + 일주동물)
├── 타고난 성향 TOP3 (십성 기반)
├── 사주 원국 (4주 그리드 + 신강신약)
├── 오행 분포 (바 차트)

├── 나는 어떤 사람인가

├── 사람들과 어울리는 방식
├── 지금 내 시기는
└── 건강과 에너지
```

## 3. 기능별 현황

### 3-1. 온보딩 ✅ 완료
- 이름(선택), 생년월일, 태어난 시간(모르면 체크), 성별 입력
- 시간 미입력 시 자정(00시, 자시) 기본값 적용 + 안내 문구 표시
- 입력값 localStorage 영구 저장 → 재방문 시 자동 복원

### 3-2. 내 사주 홈 ✅ 완료

| 섹션 | 내용 | 구현 방식 |
|------|------|----------|
| 나의 기운 | 메인 오행 + 오행 기질 설명 + 일주동물 | 백엔드 계산 |
| 타고난 성향 | 십성 TOP3 (강점/약점/환경 카드) | 백엔드 계산 + 프론트 데이터 |
| 사주 원국 | 년/월/일/시 4주 + 천간·지지·오행 표시 + 신강신약 | 백엔드 계산 |
| 오행 분포 | 목/화/토/금/수 바 차트 (비율 + 의미) | 백엔드 계산 |

### 3-3. 사주 분석 ✅ 완료

AI(Gemini 2.5 Flash)가 6개 섹션 / 23개 항목 개인화 텍스트 생성

| 섹션 | 항목 수 |
|------|--------|
| 나는 어떤 사람인가 | 핵심기질, 내면의 나, 사회적 이미지, 강점, 약점과 함정, 힘이 되는 기운 (6) |
| 일과 적성 | 일하는 스타일, 잘 맞는 분야, 직장 vs 독립, 커리어 조심할 것 (4) |
| 돈과 재물 | 돈을 대하는 방식, 재물운 흐름, 돈이 들어오는 패턴, 조심할 소비습관 (4) |
| 사람들과 어울리는 방식 | 연애 스타일, 결혼과 파트너십, 친구와 인간관계, 조심할 관계 패턴 (4) |
| 지금 내 시기는 | 현재 대운 흐름, 올해 흐름, 지금 해야 할 것, 하지 말아야 할 것 (4) |
| 건강과 에너지 | 타고난 체질, 조심할 부분, 에너지 관리법 (3) |

- 결과 localStorage 캐싱 (같은 생년월일 재방문 시 재호출 없음)
- 현재 대운 + 올해 세운 메타 정보 상단 표시

### 3-4. 오늘의 운세 ✅ 완료

| 항목 | 내용 |
|------|------|
| 오늘의 한마디 | 하루를 한 문장으로 요약 |
| 종합운 | 오늘 하루 전체 흐름 (4~5줄) |
| 애정운 | 연애·관계 (2~3줄) |
| 금전운 | 재물·금전 (2~3줄) |
| 직업·학업운 | 일·커리어·학업 (2~3줄) |
| 건강운 | 컨디션·몸 (2~3줄) |

- AI(Gemini 2.0 Flash)로 생성
- 날짜 기반 캐싱 (같은 날 재방문 시 재호출 없음)

### 3-5. 궁합 🚧 미개발

**1:1 궁합 항목 (확정)**

| 항목 | 보이는 방식 |
|------|-----------|
| 종합 궁합 점수 | 점수/등급 + 한줄 코멘트 |
| 오행 궁합 | "잘 맞아요 / 노력이 필요해요" |
| 일주 궁합 | "자연스럽게 끌리는 사이" 등 |
| 성향 궁합 | "꼼꼼한 사람 + 추진력 있는 사람" 등 |

**그룹 궁합 항목 (확정)**
- 인원: 최소 3명 ~ 최대 8명
- 그룹 전체 케미 점수 + 요약
- 구성원 간 1:1 궁합 매트릭스
- 역할 분석 (리더형, 에너지 메이커, 분위기 메이커 등)
- 보완자 분석 (상충하는 두 사람 사이를 중재하는 구성원)

**상대방 정보 저장**: localStorage / IndexedDB (이름 + 생년월일시, 닉네임 지정 가능)

### 3-6. 신년·월별운세 🚧 미개발

**신년운세**
- 연중 언제든 조회 가능 (연초 한정 아님)
- 페이지 상단 앵커 버튼으로 섹션 이동
- 구성: 총운 풀이 / 집중해야 할 것 / 주의해야 할 것 / 카테고리별 운세 (재물·애정·결혼·직업·사업·건강·인맥)

**월별운세**
- 매달 1일 갱신
- 구성: 이번 달 전체 흐름 / 카테고리별 운세 / 좋은 점 / 주의할 점 / 길일 / 흉일
- 길일·흉일: 역법 계산 기반 (신살 계산)
- 생성 방식: DB 템플릿 + AI 개인화 (하이브리드)

## 5. 광고 전략 (미구현)

### 방식: 탭 진입 시 광고 게이팅 + 2시간 프리패스
1. 탭 클릭 → "광고 보고 열기" 버튼 노출
2. 광고 시청 완료 → **2시간 프리패스** (모든 탭 자유 이용)
3. 2시간 경과 → 다시 광고 시청 필요
4. 프리패스 잔여 시간 표시 (예: "1:43 남음")

### 광고 노출 위치
- 오늘의 운세 / 궁합 / 신년·월별운세 / 사주분석 탭 진입 시
- 결과 페이지 하단 배너 (프리패스 중에도 유지)

> 토스 광고 SDK 실제 연동은 런칭 준비 단계에서 진행. 개발 시 슬롯 자리만 잡아둠.

## 6. 콘텐츠 원칙

모든 해석 문구는 아래 3단계 구조를 따름:

1. **전통 개념 명시** — 오행/십성/합충 등 원래 개념을 짧게 언급
2. **의미 해석** — "그 말은 즉슨 ~" 식으로 쉽게 풀어서 설명
3. **비유로 마무리** — 자연/일상 비유로 친근하게 마무리

> 예시: "두 분의 오행은 水(수)와 木(목)입니다. 물이 나무를 키우는 상생 관계를 뜻해요. 나무가 물을 만나 쑥쑥 자라듯, 두 분은 서로에게 좋은 영향을 주는 사이입니다."

- 전문적이되 이해하기 쉽게 (너무 캐주얼 금지)
- 전문 용어(십성, 합충 등)는 반드시 쉬운 말로 풀어서 설명
