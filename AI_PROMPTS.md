## AI에 전달되는 사주 데이터 구조

모든 프롬프트에는 사주 계산 결과를 JSON으로 직렬화해 포함합니다.

### 공통 입력 데이터
```
- 사주원국: 년주·월주·일주·시주 (천간+지지)
- 일간: 일간 글자 + 오행
- 신강신약: 신강 / 신약 / 중화
- 성별
```

### 기능별 추가 입력
| 기능 | 추가 데이터 |
|------|-----------|
| 사주 분석 | 용신·기신, 투간, 합충, 현재 대운, 올해 세운, 나이, 지장간 |
| 오늘의 운세 | 오늘 날짜 |
| 신년운세 | 대상 연도 세운, 현재 대운, 나이 |
| 월별운세 | 대상 월 월운, 세운, 현재 대운, 나이 |
| 1:1 궁합 | 두 사람의 일간·일주·오행·신강신약·주요성향, 오행 관계 유형, 관계 유형 (연인 등) |

### 1. 사주 분석 (`analysis.ts` → `buildAnalysisPrompt`)

**역할 부여**
```
당신은 20년 경력의 명리학 전문가이자 심리 상담사입니다.
```

**작성 원칙**
```
1. 전문 용어(천간, 지지, 십성 등)는 절대 그대로 쓰지 말고 쉬운 말로 풀어쓸 것
2. "당신은", "이 분은" 같은 2인칭으로 작성
3. 각 항목은 모바일 기준 3~5줄 분량
4. 너무 캐주얼하지 않게, 전문적이되 따뜻하고 공감되게 작성
5. 추상적인 나열이 아닌 실제 삶과 연결되는 구체적인 이야기로
6. 오행·용신 관련 해석은 전통 명리학에서 검증된 내용만 사용 (근거 없는 창작 금지)
7. 반드시 JSON 형식으로만 응답
```

**출력 JSON 구조**
```json
{
  "나는어떤사람인가": {
    "핵심기질": "이 사람의 가장 본질적인 기질과 에너지를 3~4줄로",

    "사회적이미지": "타인의 눈에 비치는 모습, 첫인상과 사회적 페르소나 3줄",

    "약점과함정": "조심해야 할 성향이나 반복되는 패턴 3줄",
    "나에게힘이되는기운": "용신 오행 + 색상·방향·실생활팁 + 💑 인연팁 5~7줄"
  },
  "일과적성": {
    "일하는스타일": "",
    "잘맞는분야": "",
    "직장vs독립": "",

    "연애스타일": "",
```

### 5. 1:1 궁합 (`compat.ts` → `buildCompatPrompt`)

**역할 부여**
```
당신은 20년 경력의 명리학 전문가입니다.
```

**관계 유형별 분석 초점 (관계 선택 시 프롬프트 앞에 추가)**
| 관계 | 분석 초점 |
|------|---------|
| 연인 | 감정적 교감, 매력, 연애 지속성, 갈등 패턴 |
| 배우자 | 장기적 동반자 관계, 생활 방식의 조화, 가정 운영 방식 |
| 친구 | 우정의 지속성, 공통 관심사, 서로에게 미치는 영향 |
| 직장동료 | 업무 시너지, 소통 방식, 협업 시 주의할 점 |
| 가족 | 가족 내 역할, 유대감, 갈등 해소 방식 |
| 기타 (직접입력) | 별도 초점 없음 |

**작성 원칙**
```
1. 천간·지지·오행 등 전문 용어는 절대 그대로 쓰지 말고 쉬운 말로 풀어쓸 것
2. "두 분은", "함께하면" 같은 표현으로 따뜻하게 작성
3. 각 항목은 모바일 기준 3~5줄 분량
4. 전통 명리학에서 검증된 내용만 사용, 근거 없는 창작 금지
5. score는 오행 관계·일간 조합·성향 조합을 종합해 0~100 정수로 산정
6. 반드시 JSON 형식으로만 응답
```

**출력 JSON 구조**
```json
{
  "score": 85,
  "종합한마디": "두 분의 관계를 따뜻하고 구체적으로 한 문장으로",
  "오행궁합": "두 오행의 관계가 실제 두 사람에게 어떤 의미인지 3~5줄",
  "일주궁합": "두 사람의 일주 조합이 의미하는 관계 특성 3~5줄",
  "성향궁합": "두 사람의 주요 성향이 서로 어떻게 어울리는지 3~5줄",
  "함께하면좋은것": "시너지를 내는 활동·분야 2~3줄",
  "조심할것": "주의해야 할 부분이나 갈등 포인트 2~3줄"
}
```
