## 사주 계산 차별점: 태양시(지역시) 보정

### 결정
국내 대부분의 사주 앱과 달리 **태양시(지역시) 보정을 적용**한다.
서울(동경 127°)은 한국 표준시 기준선(동경 135°)보다 서쪽에 있어 실제 태양 시간이 약 32분 느리다.
전통 사주는 시계 시간이 아닌 **실제 태양의 위치**를 기준으로 계산하는 것이 정확하다.

### 앱 내 설명 문구 (시주 입력 화면 또는 결과 화면 하단)
> **사주광은 태양시 기준으로 계산합니다**
>
> 전통 사주는 시계 시간이 아닌 실제 태양의 위치를 기준으로 합니다.
> 서울은 한국 표준시보다 태양이 약 32분 늦게 움직이기 때문에,
> 입력하신 시간에서 32분을 보정하여 더 정확한 사주를 계산해드립니다.
>
> 예) 오후 1시 31분 출생 → 실제 태양시 기준 오후 12시 59분으로 계산

## 콘텐츠 톤 & 표현 원칙 (전체 앱 공통)

모든 해석 문구는 아래 3단계 구조를 따름:

1. **전통 개념 명시** - 오행/십성/합충 등 원래 개념을 짧게 언급
2. **의미 해석** - "그 말은 즉슨 ~" 식으로 쉽게 풀어서 설명
3. **비유로 마무리** - 자연/일상 비유로 친근하게 마무리

> 예시: "두 분의 오행은 水(수)와 木(목)입니다. 물이 나무를 키우는 상생 관계를 뜻해요. 나무가 물을 만나 쑥쑥 자라듯, 두 분은 서로에게 좋은 영향을 주는 사이입니다."

- 너무 캐주얼하지 않게, 전문적이되 이해하기 쉽게
- 전문 용어(십성, 합충 등)는 반드시 풀어서 설명

## 기술 방향 (확정)

- **프론트엔드**: React + TypeScript (토스 미니앱 웹뷰)
- **백엔드**: Node.js + TypeScript
- **DB**: Supabase (PostgreSQL 기반, 무료, 웹 대시보드 제공)
- **API Key 관리**: `.env` 파일
- **프론트·백엔드 분리 구조**: `frontend/` / `backend/`

### 사주 계산 엔진
- `@fullstackfamily/manseryeok` - 만세력 + 사주팔자 기본 계산
- `lunar-typescript` - 오행·십성 상세 분석 + 길흉 판단
- 신강신약, 대운수 - 직접 구현
- Supabase DB - 사주 규칙 테이블, 운세 텍스트 템플릿 저장

#### 전체 앱 플로우

```
1. 온보딩 (최초 1회)
   └── 생년월일시 입력

2. 내 사주 홈 (메인 화면)
   ├── 메인오행
   ├── 일주동물
   ├── 타고난 성향
   ├── 년주 / 월주 / 일주 / 시주
   ├── 나의 오행 분포
   └── [광고] → "사주 분석 보러가기" 버튼

3. 사주 분석 (광고 시청 후 진입)
   ├── 탭: 내 사주
   ├── 탭: 내 성향
   └── 탭: 운세 흐름
```

#### 하단 네비게이션 (탭바)
- [x] 탭바 구성 확정 → **홈 / 오늘의 운세 / 궁합 / 신년·월별운세** (4탭)

### 2. 사주 분석
- [x] 분석 깊이 → **전통 + 대중 혼합** 확정 (요약은 쉽게, 상세는 전통 방식)
- [x] 분석 항목 확정 (아래 참고)
- [x] 결과 표현 방식 → **탭/섹션 분리** 확정
- [ ] 탭 구성 및 항목 배치 방식

#### 사주 분석 포함 항목 (확정)

| 항목 | 설명 | 해석 방식 |
|------|------|----------|
| 사주원국 | 년/월/일/시 4개 기둥, 천간+지지 8글자 | 원국 표 + 쉬운 설명 |
| 메인오행 | 일간 기준 가장 강한 오행 | "당신은 불(火)의 기운" 식 |
| 일주동물 | 일주(일간+일지) 기반 캐릭터 | 친근한 동물 캐릭터로 표현 |
| 타고난 성향 | 십성 분포 기반 (정인, 편관, 식신 등) | "타고난 성향 TOP3" 형식 |
| 오행과 십성 | 오행 비율, 십성 분포 | 그래프/차트 + 쉬운 해석 |
| 신강신약 | 일간의 강약 (뿌리, 생조 기준) | "에너지가 강한/약한 사주" |
| 사주관계 | 합/충/형/파/해 관계 | "조심해야 할 관계" 등으로 |
| 대운수 | 10년 단위 대운 흐름 | 출생~90세 전체 타임라인, 과거/현재/미래 구분 표시 |

### 3. 오늘의 운세
- [x] 운세 갱신 주기 → **자정 기준**, 미리 생성 후 캐싱
- [x] 운세 카테고리 확정

| 카테고리 | 분량 | 내용 |
|---------|------|------|
| 종합운 | 4~5줄 | 오늘 하루 전체 요약 |
| 애정운 | 2~3줄 | 연애/관계 |
| 금전운 | 2~3줄 | 돈/재물 |
| 직업·학업운 | 2~3줄 | 일/커리어/학업 |
| 건강운 | 2~3줄 | 컨디션/몸 |
- [x] 운세 생성 방식 → **하이브리드** (DB 템플릿 기반 + AI 개인화 생성)
  - DB에 일간×날짜 패턴별 뼈대 문구 저장
  - AI가 템플릿 참고 + 사용자 사주 정보로 개인화
  - 자정에 미리 생성 → 캐싱 → 즉시 응답

### 4. 궁합 보기
- [x] 1:1 궁합 항목 확정

| 항목 | 전문 개념 | 사용자에게 보이는 방식 |
|------|----------|----------------------|
| 종합 궁합 점수 | 전체 궁합 | 점수/등급 + 한줄 코멘트 ("불꽃 튀는 사이" 등) |
| 오행 궁합 | 오행 상생/상극 | "잘 맞아요 / 노력이 필요해요" 등 쉬운 멘트 |
| 일주 궁합 | 일지 합·충 관계 | "자연스럽게 끌리는 사이" 등 쉬운 멘트 |
| 성향 궁합 | 십성 기반 성격 조합 | "꼼꼼한 사람 + 추진력 있는 사람" 등 쉬운 표현 |

- [x] 그룹 궁합 확정 → **1차 배포 이후 신규 기능으로 오픈 예정**

  - **인원**: 최소 3명 ~ 최대 8명
  - **결과 구성**:
    1. 그룹 전체 케미 점수 + 요약
    2. 구성원 간 1:1 궁합 매트릭스 (잘 맞는 쌍 / 안 맞는 쌍 표시)
    3. 역할 분석 (리더형, 에너지 메이커, 분위기 메이커 등)
    4. **보완자 분석** - 상충하는 두 사람 사이를 중재·보완해주는 구성원 표시
       (오행 기준: 상극 관계의 두 오행을 이어주는 오행을 가진 사람)

- [x] 상대방 정보 저장 → **로컬 저장 확정** (localStorage / IndexedDB)
  - 저장된 사람 목록에서 바로 선택 가능
  - 그룹 궁합 시 목록에서 불러와서 구성 가능
  - 이름 + 생년월일시 저장 (닉네임 지정 가능)

### 5. 신년운세 / 월별운세

#### 신년운세
- [x] 연중 언제든 조회 가능 (연초 한정 아님)
- [x] 페이지 상단에 앵커 버튼 (해시태그/버튼형) → 클릭 시 해당 섹션으로 스크롤 이동
- [x] 구성 항목 확정:
  1. 총운 풀이
  2. 집중해야 할 것 (좋은 운이 있는 부분)
  3. 주의해야 할 것
  4. 카테고리별 운세: 재물운 / 애정운 / 결혼운 / 직업운 / 사업운 / 건강운 / 인맥운

#### 월별운세
- [x] 매달 1일 갱신
- [x] 구성 항목 확정:
  1. 이번 달 전체 운세 (월간 큰 흐름)
  2. 카테고리별: 애정운 / 금전운 / 직업·학업운 / 건강운
  3. 이번 달 좋은 점
  4. 주의해야 하는 점
  5. 길운이 들어오는 날 (길일)
  6. 조심해야 하는 날 (흉일)
- [x] 오늘의 운세와 차별점 → 월간 큰 흐름 중심 (일별 디테일 아님)
- [x] 생성 방식 확정:
  - 운세 텍스트 → 하이브리드 (DB 템플릿 + AI 개인화 생성)
  - 길일·흉일 → 역법 계산 기반 (신살 계산, 정확도 우선)

### 6. 광고 전략
- [x] 광고 노출 방식 확정 → **탭 진입 시 광고 게이팅 + 2시간 프리패스**

  - 프리패스 잔여 시간 표시 (예: "2시간 자유이용권 1:43 남음")
- [x] 광고 노출 위치:
  - 각 탭 진입 시 (오늘의 운세 / 궁합 / 신년·월별운세 / 사주분석)

#### 궁합 페이지 개발 방향 (확정)

**1단계: 1:1 궁합 먼저 개발 → 이후 그룹 궁합**

**화면 흐름:**
```
궁합 탭 진입
└── 상대방 정보 입력 (이름, 생년월일시, 성별)
    └── 궁합 결과 화면
        ├── 종합 궁합 점수
        ├── 오행 궁합
        ├── 일주 궁합
        └── 성향 궁합
```

**계산 방식: 하이브리드 (MVP → 비용 최적화 순서)**
- MVP: 사주 계산(만세력 엔진) + 해석(Gemini AI) — 빠른 개발
- 이후 전환: DB 룰 기반 텍스트로 교체 (AI 비용 0원)
  - 오행 25가지 조합 텍스트 → Supabase DB
  - 일주 궁합 패턴 텍스트 → Supabase DB
  - 성향 궁합 십성 조합 텍스트 → Supabase DB

**전환 이유:** 토스 미니앱 특성상 사용량이 많아지면 Gemini API 비용 부담이 커질 수 있음.
런칭 전 DB 텍스트 준비 완료 후 AI → DB 전환 예정.
